Chapter 01
PBR Material Model

Physically Based Rendering (PBR) is the foundation of realistic 3D. Understanding each property gives you exact control over how light interacts with every surface.

🎨
color / baseColor
The raw albedo — what color the surface IS before any light hits it

This is not the final rendered color — it is multiplied by lighting. A pure white color (#ffffff) reflects 100% of incoming light. A pure black (#000000) absorbs everything.

Rule: Never use pure white or pure black as a material color in PBR — they are physically impossible for real surfaces. Keep within #111111–#eeeeee.
material.color.set(0xa06941); // warm cognac leather
🪨
roughness
Controls how microscopically smooth or jagged the surface is

0.0 = perfect mirror (chrome). 1.0 = fully diffuse (chalk). This is the single most important property for leather realism.

Mirror
0.82
ValueLooks Like
0.0–0.2Mirror, polished metal, patent leather
0.3–0.5Polished leather, lacquered wood
0.6–0.75Semi-gloss leather, smooth skin
0.76–0.88Natural leather — target zone
0.89–0.95Rough suede, worn leather
0.96–1.0Chalk, unfinished concrete
⚡
metalness
Is the surface a conductor (metal) or insulator (everything else)?

Real-world PBR uses only two values: 0 (non-metal) or 1 (pure metal). Fractional values are for surfaces that are partially metallic, like brushed anodised aluminium.

Never set metalness above 0.05 for leather, rubber, or plastic. It produces physically incorrect renders and makes them look like painted chrome.
material.metalness = 0; // leather is always 0
🌍
envMapIntensity
How much the IBL environment (HDRI/RoomEnvironment) reflects in the surface

Even with roughness = 1.0, the IBL still provides diffuse ambient fill. This property scales how visible those reflections are as specular highlights.

ValueEffect on Leather
0Zero IBL reflection — only directional lights
0.1–0.25Subtle ambient sheen — most leather products
0.3–0.5Visible IBL — polished or oiled leather
0.5+Too reflective for natural leather
For your sandal: 0.18–0.25 is the sweet spot. Gives warmth without looking wet.
🗻
normalMap / normalScale
Simulates surface micro-geometry (bumps, grain) without adding polygons

A normal map stores surface angle data as RGB colors (X=R, Y=G, Z=B). Three.js uses normalScale as a Vector2 to control intensity.

material.normalScale.set(0.55, 0.55);
// 0 = flat, 1 = full baked depth
// leather: 0.4–0.7 is natural
// higher = looks embossed / artificial
If the baked GLB normals look too strong at 1.0, pull normalScale down to 0.4–0.6 rather than removing the map entirely.
🗺️
roughnessMap / metalnessMap
Texture maps that vary roughness/metalness per-pixel across the surface

GLB files often bake roughness maps inside. They can override your code-set roughness value per-pixel. This is why clearing them gives you full control.

// Force full control — override baked maps
material.roughnessMap  = null;
material.metalnessMap  = null;
material.roughness    = 0.82; // now guaranteed
If you set roughness = 0.82 but the material still looks glossy, a baked roughnessMap from the GLB is likely overriding it.
Chapter 02
Leather Material Rules

Leather is one of the hardest materials to replicate in 3D. These are the non-negotiable rules derived from studying real leather under studio conditions.

The 7 Laws of Realistic Leather

Law 1 — Roughness is between 0.76 and 0.88 for new leather

Polished calfskin sits around 0.76. Full-grain camel leather is around 0.82. Worn or suede leather reaches 0.92+. Never go below 0.6 unless it is patent leather.

Law 2 — Metalness is always 0

Leather is an organic insulator. Any metalness above 0 will produce specular color shift (tinted highlights) which makes it look like metallic paint. Zero, always.

Law 3 — The color you set is NOT the color you see

The rendered leather color is your baseColor multiplied by all incoming light. Under a warm key light your #A06941 cognac may render lighter and more orange. Always judge color under your actual scene lighting, not in isolation.

Law 4 — Grain texture multiplied, not added

Real leather grain darkens valleys and brightens ridges. A multiply composite operation on a procedural grain texture is physically correct — it preserves the base color while adding surface variation.

Law 5 — Different parts of the sandal are different leathers

Upper strap, insole, and midsole are all different finishes and should have different roughness values. Upper strap (~0.82) is smoother and more finished than the footbed (~0.95) which has a more natural, unfinished look.

Law 6 — Never use roughnessMap + a baked GLB without clearing it first

If the 3D artist baked a roughness map into the GLB, it will override your code values per pixel. Clear it with material.roughnessMap = null before setting roughness numerically, or you will fight the file all session.

Law 7 — IBL environment must be warm-tinted for leather to look right

A neutral white HDRI environment creates cold, grey shadows on brown leather. Use a warm RoomEnvironment or warm-tinted hemisphere light. Cold ambient fill is the #1 cause of leather looking "3D-ish" instead of photographic.

Chapter 03
Lighting Architecture

Studio photography uses 3–4 light sources. Your 3D scene should mirror this. Every light has a job — if you remove one the render breaks.

The 4-Light Studio Rig (your current setup)

☀️
Key Light — DirectionalLight
The main studio light. Creates primary shadows and highlights.
new THREE.DirectionalLight(0xfffbf2, 2.0)
position.set(3.6, 5.4, 4.6) // upper-right-front
castShadow = true

Job: Defines the main light direction. Warm white #FFFBF2 replicates studio tungsten. Position is upper-right so shadows fall left — matches product photography convention.

Intensity 1.8–2.2 for leather. Lower if the highlight is blowing out. Higher if the shoe looks flat.
🌤️
Fill Light — DirectionalLight
Softens the shadow side so it isn't pure black.
new THREE.DirectionalLight(0xf4dfb7, 0.38)
position.set(-4.2, 2.2, 2.6) // left side

Job: Without this, the shadow side of the shoe goes pitch black. A warm golden fill (#F4DFB7) keeps shadow areas readable and warm.

Fill ratio = fill / key intensity. 0.38 / 2.0 = ~1:5 ratio. Studio photography typically uses 1:4–1:6. Never go above 1:2 or you lose all shadow depth.
💡
Rim Light — DirectionalLight
Separates the subject from the background with an edge highlight.
new THREE.DirectionalLight(0xffffff, 0.72)
position.set(-2.6, 3.8, -3.8) // back-left

Job: Back-left position catches the rear edges of the sandal — the strap curves and the platform edge — with a bright line that makes the shoe pop off the white background.

Cool white rim light against warm key light creates depth contrast. Keep intensity 0.5–0.85.
🌈
Hemisphere Light — Ambient Fill
Simulates sky bounce — the most important "invisible" light.
new THREE.HemisphereLight(
  0xfff5e8, // sky color (top-down warm fill)
  0xcaa77a, // ground bounce (warm tan from below)
  1.1       // intensity
)

Job: Simulates ambient environment — skylight from above and ground bounce from below. Without this, any surface not facing a directional light goes completely black.

Do not use a neutral grey hemisphere light on brown leather — it creates cold, lifeless shadows. Always warm both sky and ground colors.

IBL — Image Based Lighting

The RoomEnvironment generates a PMREM (Pre-filtered Mip-mapped Radiance Environment Map) that adds physically correct ambient reflections. Even with envMapIntensity = 0 on materials, assigning it to scene.environment provides better diffuse shading than hemisphere light alone.

// Generate and assign IBL
const pmrem = new THREE.PMREMGenerator(renderer);
const roomEnv = new THREE.RoomEnvironment();
scene.environment = pmrem.fromScene(roomEnv, 0.015).texture;
// 0.015 = environment blur — higher = softer/more diffuse reflections

Chapter 04
Color Science for 3D

Color in 3D is not the same as color in Photoshop. Lighting, tone mapping, and color spaces all transform your hex values before they reach the screen.

Color Space — The Most Common Mistake

Always set outputColorSpace to SRGBColorSpace

Three.js works in linear light internally. Without SRGB output, colors look washed out and oversaturated in shadows. This single line fixes ~30% of "why does it look wrong" questions.

renderer.outputColorSpace = THREE.SRGBColorSpace; // mandatory

Trick: To judge your actual material color, temporarily set all light intensities to 1.0 and exposure to 1.0. Then tune the hex values. Restore lighting after.

Warm vs Cool Color Bias

Brown leather under warm (yellow/orange) light looks rich and natural. Under cool (blue) light the same leather looks desaturated and artificial. Your key light #FFFBF2 has a slight warm bias — this is intentional and matches studio tungsten.

Chapter 05
Procedural Leather Textures

Your component generates leather grain programmatically using Canvas 2D. This removes any dependency on texture files and gives you full code control.

How the Grain Texture Works

🎲
Layer 1 — Noise Base
Per-pixel random variation over the base color
// Seeded random so grain is consistent on reload
const variation = (random() - 0.5) * config.noise;
image.data[i]   = clamp(baseR + variation);
image.data[i+1] = clamp(baseG + variation);

noise: Controls how much the pixel-level color varies. 38 for leather (visible grain), 22 for footbed (smoother).

〰️
Layer 2 — Grain Lines
Curved brushstroke-like lines simulating leather fibres
context.globalCompositeOperation = 'multiply';
// Quadratic Bézier curves at slight angles
// grainLines: 1500 = dense (upper leather)
// grainLines: 520  = sparse (footbed)

Multiply blending darkens only grain valleys — matching how real leather grain catches shadows.

⚫
Layer 3 — Pores
Small dark ellipses simulating leather pore structure
// Randomised ellipses at varying angles
context.ellipse(x, y, rx, ry, angle, 0, Math.PI*2);
// pores: 3800 for leather, 1800 for footbed
// poreAlpha: 0.12 → subtle, not spotty

Higher pore count = more visible grain texture. Camel leather has larger, sparser pores than calfskin.

✨
Layer 4 — Highlight Lines
Screen-mode bright lines simulating leather ridge catches
context.globalCompositeOperation = 'screen';
// 22% of grain line count
// rgba(255,236,208,...) warm white highlights
// Very subtle: 0.03–0.11 alpha range

Screen mode brightens ridge tops, completing the illusion of real leather grain depth.

Texture Settings (Three.js)

texture.colorSpace = THREE.SRGBColorSpace; // match renderer output
texture.wrapS = texture.wrapT = THREE.RepeatWrapping; // tile it
texture.repeat.set(2.35, 2.35);  // tile density — higher = finer grain
texture.anisotropy = 4;           // sharper at glancing angles
texture.needsUpdate = true;
Chapter 06
Tone Mapping

Tone mapping converts HDR linear light values to the limited 0–1 SDR range of a screen. Choosing the right operator and exposure dramatically affects the final look.

Tone Mapping Operators

OperatorLookBest For
LinearToneMappingNo processing — raw linear valuesDebug only — looks washed out
ReinhardToneMappingSoft roll-off, preserves huesNeutral product renders
ACESFilmicToneMappingCinematic — warm shadows, rich highlightsLeather / luxury products ← YOUR CHOICE
CineonToneMappingFilm-print look, warm contrastVintage / editorial feel
AgXToneMappingNeutral, preserves hue under exposureClinical product renders

Exposure Scale

renderer.toneMappingExposure = 1.05;
// This is a MULTIPLIER on the entire scene's light energy
// 0.70 → dark studio    0.85 → normal    1.05 → bright/airy    1.30 → overexposed
Technique: Set exposure last, after all colors and lights are dialed in. It's the master brightness knob — one number fixes overall darkness/brightness without touching individual lights.

GLB Export Pipeline

The quality of your 3D render starts in the modelling software, not in the code. These are the rules for exporting GLB files that work well with your Three.js renderer.

Blender Export Settings (Recommended)

# File → Export → glTF 2.0 (.glb/.gltf)

Format: GLB (binary)
Include:
  ✓ Selected Objects
  ✓ Apply Modifiers
  ✓ UVs
  ✓ Normals
  ✓ Tangents              # needed for correct normal maps
  ✗ Vertex Colors         # skip — use material colors instead

Transform:
  ✓ Y Up                  # Three.js uses Y-up

Geometry:
  ✓ Apply Modifiers
  ✗ Loose Edges / Points

Compression:
  ✓ Draco Mesh Compression  # reduces file size ~70%
  Compression Level: 6      # balanced quality/size

Chapter 09
Common Problems & Fixes

Every problem you have encountered and the exact fix for each one.

🌑
Model looks too dark
Underexposed, shadow areas are black

Causes (in order of likelihood):

  1. Exposure too low — raise toneMappingExposure
  2. Hemisphere light too dim — raise intensity above 1.0
  3. IBL removed from scene — restore scene.environment
  4. Colors set too dark — the hex value itself is too dark
// Quick fix
renderer.toneMappingExposure = 1.05; // was 0.78
new HemisphereLight(..., ..., 1.1);  // was 0.74
✨
Model looks too glossy / reflective
Specular highlights too strong, looks wet

Causes:

  1. Roughness too low — raise toward 0.88+
  2. envMapIntensity too high — lower toward 0–0.15
  3. roughnessMap from GLB lowering roughness in patches
  4. metalness > 0 — set to 0
material.roughness      = 0.88; // raise
material.roughnessMap   = null; // clear baked map
material.envMapIntensity = 0.12; // lower
🎨
Colors look wrong / too different from photo
Hex value doesn't match what renders on screen

Causes:

  1. outputColorSpace not set to SRGBColorSpace
  2. Warm key light tinting the color orange
  3. Exposure multiplying the color lighter
  4. Grain texture multiply-blending shifting the color
renderer.outputColorSpace = THREE.SRGBColorSpace;
// If still wrong: temporarily set exposure 1.0
// and all light intensities to 1.0, then tune hex
📦
Material color not changing after code edit
Updated hex but the model still shows old color

Causes:

  1. Missing material.needsUpdate = true after change
  2. GLB has a baked color texture (material.map) overriding material.color
  3. Material name doesn't match detection patterns in code
material.color.set(newColor);
material.needsUpdate = true; ← required

// If baked texture overrides color:
material.map = null; // clears the baked albedo
🥶
Leather looks cold / grey / plastic
Correct color but wrong "feel"

Causes:

  1. Hemisphere light sky color is white/grey — make it warm
  2. Key light color is pure white — shift to #FFFBF2
  3. No fill light — shadows go cold grey
  4. IBL RoomEnvironment too neutral
// Warm hemisphere — fixes cold shadow problem
new HemisphereLight(
  0xfff5e8, // warm sky (not 0xffffff)
  0xcaa77a, // tan ground bounce
  1.1
);
🔤
New model material not picking up color
Added a new GLB but materials ignore code colors

Causes:

  1. Material name in GLB doesn't match any detection pattern
  2. Material is not MeshStandardMaterial type — must convert in Blender
  3. Material has a baked albedo texture that overrides color
// Debug: log all material names on load
model.traverse(child => {
  if (child instanceof THREE.Mesh) {
    const mats = Array.isArray(child.material)
      ? child.material : [child.material];
    mats.forEach(m => console.log(m.name));
  }
});
Chapter 10
Production Checklist

Run through this before considering any 3D model or material change complete.

Renderer Setup

  • outputColorSpace = SRGBColorSpace is set
  • toneMapping = ACESFilmicToneMapping
  • toneMappingExposure is between 0.85–1.15
  • shadowMap.enabled = true
  • pixelRatio capped at 2 (performance)
  • Canvas resizes correctly with ResizeObserver

Materials

  • All leather parts: metalness = 0
  • Upper leather: roughness between 0.76–0.88
  • Footbed: roughness above 0.93
  • Buckle: roughness above 0.90
  • roughnessMap = null to override GLB baked maps
  • material.needsUpdate = true after any change
  • All material names checked in console log

Lighting

  • Key light: warm white, intensity 1.6–2.2
  • Fill light: warm gold, intensity 0.3–0.5
  • Rim light: white, intensity 0.6–0.9, back position
  • Hemisphere: warm sky AND warm ground colors
  • IBL (scene.environment) is assigned
  • Key light has castShadow = true
  • Shadow map size ≥ 1024 (2048 for production)

GLB File

  • Exported with Draco compression enabled
  • File size under 1.5 MB
  • All material names match detection patterns
  • Materials are MeshStandardMaterial type in Blender (Principled BSDF)
  • Y-up coordinate system selected in export
  • DRACOLoader decoder path set to /assets/draco/
  • Model loads and traversal runs without errors

Color Accuracy

  • Color compared against product photography, not against hex swatch
  • Checked on both light and dark monitors
  • Leather swatch color and swatch fields both updated
  • Arabic and English i18n strings added for new swatches
  • Default selected color (cognac) matches product photography baseline

Performance

  • Animation runs with requestAnimationFrame
  • Frame cancelled in ngOnDestroy
  • ResizeObserver disconnected on destroy
  • Textures disposed on model clear
  • Only one DRACO loader instance per load cycle
