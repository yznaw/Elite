Storefront Integration Brief
Feedback System —
What you need to build
Elite Collection has a private feedback system that collects reviews from customers across three surfaces. Two are already live. Your surface — the storefront product page — is the one remaining piece.
iPad Kiosk Live
Fullscreen multi-step form on in-store iPad. Bilingual EN/AR. Submits to POST /api/products/:id/reviews or general POST /api/reviews.
Admin Portal Live
Elite team reads all feedback at /feedback in the admin portal. Shows product ratings, review text, author contact info. General (non-product) feedback has its own tab.
Storefront Product Page Your job
A form — or collapsible section, or floating bar — on the product page that lets any visitor (buyer or not) leave a star rating and/or comment. Completely private, never shown publicly.
Backend API Ready
Two endpoints accept submissions. No auth token required — open to any visitor. source: 'storefront' tags storefront reviews so the team can filter by channel.
Key rules to know
◇
Feedback is always private It is never shown publicly on the storefront. Do not build a public review display section. All submissions go straight to the admin portal for the team to read.
ℹ
No purchase required Any visitor can submit — browsers, gift buyers, returning customers, first-timers. The form must not be gated behind login or purchase status.
ℹ
All fields are optional except the product ID Rating and body are both optional. Name, phone, and email are always optional. A valid submission can be just a star with nothing else — or just a body with no rating.
◇
Set source: 'storefront' Always include this field in your POST body. The admin team uses it to distinguish storefront feedback from kiosk feedback.
Minimum required deliverable
1
A way to submit feedback from every product pagePick one of the 4 design patterns in the Design Options tab. The implementation details (stack, component name, placement) are yours to decide — the API shape is fixed.
2
Read the product ID from the page contextEach product page has a UUID (e.g. from the route params or page data). Pass it in the URL as POST /api/products/<UUID>/reviews.
3
Show a success / thank-you state after submitNo redirect needed. A small inline confirmation ("Thank you — your thoughts reach our artisans personally.") is enough. The form should reset if the user wants to submit again.
4
Mobile-first tap targetsAll interactive elements must be ≥ 44 px tall on mobile. The form must work on phone browsers without horizontal scroll.
API Reference
Two endpoints.
No auth needed.
Both endpoints are unauthenticated — any browser can call them. The tenant is resolved server-side from the request origin, so you don't need to pass a tenant ID.
Base URL
/* same origin as the storefront — no cross-origin needed */ const API = '/api'; // production const API = 'http://localhost:3000/api'; // dev (server on 3000)
POST /api/products/:productId/reviews Product feedback

Submit feedback linked to a specific product. Use this on the product detail page where you know the product ID. At least one of rating or body must be present.

Request body (JSON)
FieldTypeRequiredNotes
ratinginteger 1–5OptionalStar rating. Omit or pass null if the user didn't rate.
bodystringOptionalFree text. Omit or pass null if the user left nothing.
authorNamestringOptionalCustomer name.
authorPhonestringOptionalMobile number. No format enforced by the API.
authorEmailstringOptionalEmail address.
sourcestringRequiredAlways pass "storefront" — used for channel tracking.
// Example — product page, user rated 4 stars + left a message const productId = 'a1b2c3d4-...'; // from page/route context await fetch(`/api/products/${productId}/reviews`, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ rating: 4, body: 'Incredible craftsmanship — the ostrich leather is stunning.', authorName: 'Fatima Al-Sayed', authorPhone: '+974 5555 1234', authorEmail: null, source: 'storefront', }), }); // Success → 201 Created // { "success": true, "data": { "id": "...", "createdAt": "..." } } // Error → 400 Bad Request (if both rating AND body are null/missing)
POST /api/reviews General feedback (no product)

Submit feedback not linked to any product — useful for a general "Contact / Feedback" section, a footer form, or a standalone page. Same fields as above, no productId in URL.

// Example — general feedback form, just a message, no rating await fetch('/api/reviews', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ rating: null, body: 'Wonderful experience, very helpful staff.', authorName: 'Ahmed', authorPhone: null, authorEmail: null, source: 'storefront', }), });
HTTP Status Codes
StatusMeaningWhen
201 CreatedSuccessReview saved. Show your thank-you state.
400 Bad RequestValidation errorBoth rating and body are missing or null.
404 Not FoundBad product IDThe :productId UUID doesn't match any product in the tenant.
500Server errorUnexpected. Retry once, then fail silently — don't block the user.

Implementation Guide
Step-by-step
No SDK required — plain fetch, one POST call, one JSON body. Works in any framework.
1
Get the product ID from your page contextYour storefront routes likely have the product slug or UUID in the URL or page data. Extract the UUID to use as the path param. If your system uses slugs, look up the UUID from your product data source before submitting.
2
Decide which design pattern to use (see Design Options tab)Option A (accordion) requires the least new component work — it can be an additional row appended to the existing accordion list. Option B (sticky bar) requires a scroll listener and an overlay.
3
Build the form with these fieldsRating (1–5, optional), body/message (textarea, optional), name (optional), phone (optional), email (optional). Require at least one of rating or body — show a small inline validation message if both are empty.
4
Submit with fetchSee the code example below. Disable the submit button while the request is in flight. On 201 → show success state. On any other status → show a small inline error ("Something went wrong — please try again").
5
Success stateReplace the form with a short thank-you message. Suggested copy: "Thank you — your words reach our artisans personally." A small gold checkmark icon looks great here. Do not redirect the user.
Complete example (vanilla JS)
/** * Submit product feedback from the storefront. * Call this from your form's submit handler. */ async function submitFeedback({ productId, rating, body, name, phone, email }) { // Validate — at least one of rating or body required if (!rating && !body?.trim()) { throw new Error('Please add a rating or a message before submitting.'); } const endpoint = productId ? `/api/products/${productId}/reviews` : '/api/reviews'; const res = await fetch(endpoint, { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ rating: rating || null, body: body?.trim() || null, authorName: name?.trim() || null, authorPhone: phone?.trim() || null, authorEmail: email?.trim() || null, source: 'storefront', }), }); if (!res.ok) { const err = await res.json().catch(() => ({})); throw new Error(err.message || 'Submission failed'); } return await res.json(); // { success: true, data: { id, createdAt } } } // Usage in a form submit handler formEl.addEventListener('submit', async (e) => { e.preventDefault(); submitBtn.disabled = true; submitBtn.textContent = 'Sending…'; try { await submitFeedback({ productId: '<from page context>', rating: selectedRating, // integer 1–5 or null body: bodyInput.value, name: nameInput.value, phone: phoneInput.value, email: emailInput.value, }); showThanksState(); } catch (err) { errorEl.textContent = err.message; submitBtn.disabled = false; submitBtn.textContent = 'Submit →'; } });
Edge cases to handle
Network offline
Wrap the fetch in try/catch (shown above). Show inline error — never let a network failure break the whole product page. The form should still be usable once they reconnect.
Double-submit
Disable the submit button while the request is in flight (submitBtn.disabled = true). Re-enable on error only — on success, replace the form with the thanks state.
RTL / Arabic pages
If the storefront supports Arabic, set dir="rtl" on the form container and mirror any directional arrow icons. The API accepts Arabic text without special handling.
Rating-only submission
A tap on a star with no message is valid — pass body: null. The API accepts it. Don't force the user to type anything.
◇
Questions? Contact the Elite Collection admin team. The backend is already running — your only dependency is the API base URL (same origin in production, http://localhost:3000 in local dev). No credentials, no API keys, no CORS issues.
