1 — Server Setup
Provision a VPS and install required software
Critical
Provision a VPS (Ubuntu 22.04 recommended)
DigitalOcean, Hetzner, or AWS Lightsail. Minimum 1 GB RAM / 1 vCPU for this app. Add your SSH key during setup.
Ubuntu 22.04 LTS1 GB+ RAM1 vCPU+
Install Node.js 20 LTS
Use the NodeSource setup script — do NOT use the Ubuntu default node which is outdated.
# Install Node 20 LTS via NodeSource curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt install -y nodejs node -v # should print v20.x.x
Install PostgreSQL 15+
Install Postgres, create a dedicated database user and database for the app.
sudo apt install -y postgresql postgresql-contrib sudo -u postgres psql -c "CREATE USER elite WITH PASSWORD 'STRONG_PASSWORD_HERE';" sudo -u postgres psql -c "CREATE DATABASE elite OWNER elite;"
Install Nginx
sudo apt install -y nginx sudo systemctl enable nginx
Install PM2 (process manager)
PM2 keeps the Node server running and auto-restarts it on crash or reboot.
sudo npm install -g pm2 pm2 startup # follow the printed command to enable on boot
Configure UFW firewall
Allow only SSH, HTTP (80), and HTTPS (443). Block direct access to port 3000.
sudo ufw allow OpenSSH sudo ufw allow 'Nginx Full' sudo ufw enable

2 — Deploy the Code
Get the codebase onto the server
Critical
Clone the repository onto the server
cd /var/www sudo git clone https://github.com/YOUR_ORG/elite.git sudo chown -R $USER:$USER /var/www/elite
Install server dependencies
cd /var/www/elite/server && npm install --omit=dev
Build Angular admin portal (production build)
Run this on the server (or locally and upload the dist/ folder). Output goes to client/dist/admin-portal/browser/.
cd /var/www/elite/client npm install npx ng build admin-portal --configuration=production
Create the uploads / media storage directory
The server saves uploaded images here. Must be writable by the Node process.
mkdir -p /var/www/elite/server/uploads chmod 755 /var/www/elite/server/uploads

3 — Environment Variables
Production secrets — never commit these to git
Critical
⚠️
Never commit .env to gitThe .env file must be created manually on the server. It should NOT be in the repository. Verify .gitignore includes server/.env.
Create server/.env with production values
Copy the template and fill in every value. Use a strong random SESSION_SECRET (32+ chars).
# /var/www/elite/server/.env PORT=3000 NODE_ENV=production # PostgreSQL — use the user/db you created in step 1 DATABASE_URL=postgresql://elite:STRONG_PASSWORD@localhost:5432/elite DEFAULT_TENANT_SLUG=elite DEFAULT_TENANT_NAME=Elite DEFAULT_CURRENCY=QAR # Elite production domains CORS_ORIGINS=https://elitecollections.qa,https://www.elitecollections.qa,https://admin.elitecollections.qa # Generate with: openssl rand -hex 32 SESSION_SECRET=REPLACE_WITH_64_CHAR_RANDOM_STRING SESSION_COOKIE_NAME=elite.sid SESSION_MAX_AGE_MS=43200000 SESSION_COOKIE_SECURE=true # MUST be true with HTTPS SESSION_COOKIE_SAMESITE=lax # Google Drive image import (optional) GOOGLE_API_KEY=YOUR_GOOGLE_API_KEY # Default admin — change password immediately after first login! DEFAULT_ADMIN_EMAIL=admin@elitecollections.qa DEFAULT_ADMIN_PASSWORD=CHANGE_ME_IMMEDIATELY DEFAULT_ADMIN_NAME=Admin
Generate a strong SESSION_SECRET
openssl rand -hex 32
Copy the output and paste it as the SESSION_SECRET value.
Restrict .env file permissions
chmod 600 /var/www/elite/server/.env

4 — Database Migrations
Create all tables on the production database
Critical
Run initial schema migration (001)
Creates all core tables: products, variants, media_assets, orders, etc.
psql "postgresql://elite:PASSWORD@localhost:5432/elite" \ -f /var/www/elite/server/db/migrations/001_initial_schema.sql
Run reference tables migration (003)
Creates ref_colors, ref_materials, ref_size_sets tables.
psql "postgresql://elite:PASSWORD@localhost:5432/elite" \ -f /var/www/elite/server/db/migrations/003_ref_tables.sql
Seed the default tenant + reference data
The server seeds the default tenant and admin user on first boot automatically when DEFAULT_TENANT_SLUG is set. You can also run the seed script manually.
cd /var/www/elite/server node index.js # start once — it seeds on boot, then Ctrl+C and use PM2
Verify tables were created successfully
psql "postgresql://elite:PASSWORD@localhost:5432/elite" -c "\dt" # Should list: products, product_variants, media_assets, # media_links, ref_colors, ref_materials, ref_size_sets, # tenants, admin_users, sessions, orders, ...

5 — PM2 Process Manager
Keep Node running in the background, auto-restart on crash and reboot
Critical
Start the API server with PM2
cd /var/www/elite/server pm2 start index.js --name elite-api --env production pm2 save # persist so it survives reboots
Verify PM2 shows the server as "online"

Confirm health endpoint responds

6 — Nginx Configuration
Serve Angular, terminate HTTPS, and proxy API requests
Critical
Create Nginx site config for the admin portal
Use admin.elitecollections.qa for the admin subdomain. Express stays private on 127.0.0.1:3000; Nginx is the public web server.
# /etc/nginx/sites-available/elite-admin upstream elite_api { server 127.0.0.1:3000; } server { listen 80; server_name admin.elitecollections.qa; root /var/www/elite/client/dist/admin-portal/browser; index index.html; # Angular: serve index.html for all unknown routes (SPA routing) location / { try_files $uri $uri/ /index.html; } # Proxy all /api/* requests to the Node server location /api/ { proxy_pass http://elite_api; proxy_http_version 1.1; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; proxy_cache_bypass $http_upgrade; proxy_read_timeout 300s; # needed for long bulk imports } # Serve uploaded media files directly location /uploads/ { alias /var/www/elite/server/uploads/; expires 30d; add_header Cache-Control "public, immutable"; } }
Enable the site and test Nginx config
sudo ln -s /etc/nginx/sites-available/elite-admin /etc/nginx/sites-enabled/ sudo nginx -t # must say "syntax is ok" sudo systemctl reload nginx

7 — SSL / HTTPS
Let Nginx handle public HTTPS with Let's Encrypt
Critical
ℹ️
DNS must point to your server firstMake sure admin.elitecollections.qa is pointing to your server's IP in your DNS settings before running Certbot, otherwise it will fail. Node does not need HTTPS certificates; Nginx terminates TLS and proxies to Express over localhost.
Point your domain's DNS A record to the server IP
In your domain registrar's DNS settings, add an A record: admin → YOUR_SERVER_IP. Allow up to 10 minutes for propagation.
DNS propagation can take up to 10 min
Install Certbot and get SSL certificate
Certbot will automatically edit your Nginx config to add HTTPS, redirect HTTP to HTTPS, and set up auto-renewal.
sudo apt install -y certbot python3-certbot-nginx sudo certbot --nginx -d admin.elitecollections.qa # Follow the prompts — choose to redirect HTTP → HTTPS
Verify Nginx HTTPS is working
curl -I https://admin.elitecollections.qa/api/health # Should return HTTP/2 200 with {"success":true,...}
🛡️
8 — Post-Launch Security
Harden the live environment immediately after first login
Critical
Change the default admin password immediately
Log in to the admin portal and change the password from DEFAULT_ADMIN_PASSWORD to a strong unique password in Settings → Account.
Do this on first login — don't skip
Remove DEFAULT_ADMIN_PASSWORD from .env
Once the admin account exists in the DB, the DEFAULT_ADMIN_PASSWORD env var is no longer needed. Remove it to prevent accidental re-seeding.
Verify SESSION_COOKIE_SECURE=true in .env
With HTTPS live, cookies must be set to secure-only or they won't be sent by the browser.
Disable password-based SSH — use SSH keys only
# Edit /etc/ssh/sshd_config: PasswordAuthentication no sudo systemctl restart sshd

9 — Backups & Monitoring
Protect data and know when things break
Important
Set up daily PostgreSQL backups
Add a cron job to dump the database every night and keep 7 days of backups.
# Add to crontab: crontab -e 0 2 * * * pg_dump "postgresql://elite:PASSWORD@localhost:5432/elite" \ | gzip > /backups/elite-$(date +\%Y\%m\%d).sql.gz \ && find /backups -name "*.sql.gz" -mtime +7 -delete
Enable PM2 monitoring / logs
pm2 logs elite-api # tail live logs pm2 monit # real-time CPU / memory dashboard
Set up uptime monitoring (optional but recommended)
Free options: UptimeRobot or BetterUptime — monitor https://admin.elitecollections.qa/api/health and get an email/SMS if it goes down.
UptimeRobot — free tierBetterUptime — free tier
