🔴 Fix First — Currently Broken
⚙️
Settings — Save actually persists Broken
Store name, currency, timezone, logo upload — all currently show a toast but save nothing to the DB. Wire the Save button to POST /api/admin/settings and load on page open.
📥
Orders — Real CSV export Broken
The "Export CSV" button shows a toast but downloads nothing. Generate a real .csv file from the current filtered order list and trigger download — same pattern as import report already in the codebase.
📊
Analytics — Real data Broken
All analytics charts use mock.ts. Connect to the existing /api/admin/analytics endpoints which already query the real DB for revenue, orders, top products, and traffic data.
🟠 Critical — Will Break with Real Data
📄
Pagination — All admin lists Critical
Products, orders, customers, and media all load every row at once. With 500+ products or 1,000+ orders this will be slow and unusable. Add server-side pagination (page + limit params) to all 4 list endpoints and add a simple page control to each list UI.
🗑️
Orphaned Media Cleanup Do Now
There are currently 272 unlinked images from the first failed import sitting on disk and in the DB, linked to nothing. One button in the Media page: "Clean up X unlinked files (Y MB)" → confirm → delete. Prevents wasted storage and a cluttered media library.
🔵 High Value — Daily Use Features
📋
Duplicate Product New
A "Duplicate" button in the product drawer toolbar. Copies the product + all variants + description + price. SKU gets a -COPY suffix. Status set to Hidden. Opens the duplicate immediately in the drawer. Daily time-saver for a shop adding similar styles.
📤
Export Catalog to CSV New
Download the currently filtered catalog as a CSV in the same format as the import template. Useful for offline editing, price updates, and re-importing. Button in the catalog toolbar next to Import.
🖨️
Print Invoice / Packing Slip New
A "Print" button in the order drawer. Opens a clean, print-optimised view with: order number, date, customer name and address, line items with SKU/size/color/price, subtotal, total. No dependencies — pure CSS @media print. Every shop ships physical goods and needs this.
🏷️
Brands & Categories New
Right now brand is hard-coded as "Elite" and there are no product categories. Add a category picker (Men → Footwear → Slippers) in the product drawer, and a category filter in the catalog. Required for storefront navigation. Simple 2-level hierarchy — no need for unlimited depth.
🔍
Product SEO Fields New
Add a collapsible "SEO" section at the bottom of the product drawer with: Meta Title, Meta Description (with character counter), and URL Slug (auto-generated but editable). These fields exist in the DB schema but aren't surfaced in the UI yet.
📅
Dashboard Date Range Filter Need
The dashboard is fixed to "Last 30 days". Add Today / 7 days / 30 days / 90 days toggle buttons above the KPI cards. Updates all charts and metrics. One of the most-used admin features in any shop.
📦
Bulk Stock Update via CSV New
A simple "Stock Import" tab in the bulk import dialog. Upload a 2-column CSV (SKU, Stock). Updates variant stock quantities in bulk — essential after a physical inventory count. Shows SKUs not found in the report.
⚠️
Low Stock Alerts New
A "Low Stock" card on the dashboard showing how many variants are below threshold (default: 5 units). Clicking it filters the catalog to show only low-stock products. No emails — just a visible dashboard signal.
📱
Mobile Responsiveness Audit Need
QA pass at 390px (iPhone), 768px (iPad), and 1024px. Known issues: some table columns overflow, sidebar doesn't collapse on tablet, filter panel width. Fix any layout breaks. The shop owner likely checks the portal on their phone.
🔐
Role-Gated Bulk Operations Must Have
Restrict destructive bulk operations (bulk delete, bulk import, bulk status change) to Owner and Admin roles only. Managers see the catalog but bulk action buttons are hidden. Server enforces the role check on every bulk endpoint — the UI gate alone is not enough. Required once team invitations are live.
🔍
Dry-Run Import (Preview Before Commit) New
Upload a CSV and see exactly what would happen — how many products would be created vs updated, which rows have validation errors — before writing a single row to the database. A "Preview" checkbox in the import dialog. Gives confidence before importing 300+ products.
🟣 Polish & Completeness
🔄
Retry Failed Import Rows New
After an import, if some products failed (image download timeout, network error), a "Retry Failed (N)" button re-runs only those rows without re-processing everything.
📋
Import History Log New
A history tab in the import dialog showing past imports with date, filename, and counts. Click to expand and see per-product results. Re-download the CSV report from any past import.
🖼️
Image Deduplication New
Before saving a downloaded image, compute its SHA-256 hash and check if an identical file already exists. Re-use the existing asset instead of saving a duplicate. Prevents storage waste on repeated imports.
🏷️
Product Tags New
Free-form tags (Best Seller, New Arrival, Ramadan, Sale) assignable to products and usable as catalog filters. Simple multi-select in the product drawer. Useful for seasonal marketing.
🌐
Storefront — Live Split-Screen Preview New
Embed an iframe preview of the storefront alongside the editor so changes are visible in real time without opening a new tab. The editor and preview sit side-by-side.
📅
Orders — Date Range Filter New
Today / This Week / This Month / Custom date range filter on the orders list. Useful for end-of-day or end-of-month reporting.
📏
Size Column in CSV Import New
Add an optional Size column to the import template so each CSV row can represent a specific color + size combination. Allows full variant grids to be imported in one file.
👥
Team Invitations New
Invite team members by email from the Settings → Team page. Each invite generates a secure token link. The recipient sets their own password on first login. Role is assigned at invite time (admin, manager, viewer).
📱
Storefront — Mobile Preview Toggle New
Toggle the storefront preview between Desktop, Tablet (768px), and Mobile (390px) viewports using an iframe resize. Builds on top of the live split-screen preview.
