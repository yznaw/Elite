Feedback Feature · Full Plan
Three surfaces.
One feedback system.
Feedback is collected privately from three entry points: the online product page, the in-store iPad kiosk, and optionally a QR code. All data flows into a single database table and surfaces in the admin portal.
Surface 1 — Online
Storefront Product Page
A collapsible feedback bar below the product accordions — same style as "NOTIFY ME". Tap to expand → mood emoji tiles → optional text + contact details.

Who: Any visitor — browsing, buying, or just curious.
Auth: None. Public endpoint.
Shows publicly: Never. Admin-only.
Surface 2 — In-store
iPad Kiosk
A dedicated fullscreen touch app at the checkout counter. Staff hands the iPad to the customer after purchase. 5-step flow: Welcome → Rating → Message → Contact → Thank you.

Who: Walk-in customers at the physical store.
Auth: None. Kiosk is locked to a single URL.
Deployment: Subdomain + Nginx + Guided Access on iPad.
Surface 3 — Admin
Feedback Portal Pages
Two new admin pages: /feedback (all products with review counts + avg rating) and /feedback/:productId (all reviews for one product with contact info + delete).

Who: Admin team only.
Features: Copy email, delete review, average rating, latest date.

iPad Kiosk · Full Deployment Plan
From code to counter
End-to-end plan: server config → subdomain → Angular route → iPad setup. No extra server, no extra hosting cost — everything runs on the existing server.
Architecture decision: subdomain vs route
Recommended: subdomain — kiosk.elitecollections.qa
Cleaner URL for the iPad bookmark, isolated from the main storefront, easier to give to staff ("just go to kiosk.elitecollections.qa"). Requires 1 DNS A record + 1 Nginx server block + 1 SSL cert.

Alternative: route — elitecollections.qa/kiosk
Zero DNS/Nginx changes — just add a new Angular route. But shares the storefront bundle and URL is longer.
Phase 1 — Build the kiosk Angular page
New route in client-web
1
Create kiosk component
New standalone Angular component at client/projects/client-web/src/app/pages/kiosk/kiosk.component.ts. Self-contained fullscreen template — no nav, no footer. Handles the 4-step flow internally with signals.
2
Add route
In app.routes.ts: { path: 'kiosk', loadComponent: () => import('./pages/kiosk/kiosk.component') }. Lazy-loaded so it doesn't bloat the main bundle.
3
Add source tag to submit endpoint
The review form already hits POST /api/products/:id/reviews. The kiosk passes source: 'kiosk' in the body; the server stores it. The storefront passes source: 'storefront'. DB column added: source text DEFAULT 'storefront'.
4
Auto-reset timer
On the "Thank you" screen, start a 10-second countdown. If no tap in 60 seconds on any screen, reset to Welcome. Use fromEvent(document, 'touchstart') to detect activity and reset the timer. Standard kiosk pattern.
5
PWA manifest entry (optional but recommended)
Add to manifest.webmanifest: display: "standalone", start_url: "/kiosk", custom theme colour. Staff can then "Add to Home Screen" on the iPad and it launches fullscreen with no browser UI.
Phase 2 — Server config (subdomain approach)
DNS + Nginx + SSL
6
Add DNS A record
In your DNS provider (Cloudflare / GoDaddy / etc.) add:
Type: A   Name: kiosk   Value: [same server IP as elitecollections.qa]   TTL: Auto
Takes 1–30 minutes to propagate. Can test with dig kiosk.elitecollections.qa.
7
Nginx server block for kiosk subdomain
Add to /etc/nginx/sites-available/elite (or create a new file). The kiosk subdomain serves the same Angular build but with a custom Content-Security-Policy that prevents navigation away from /kiosk:

server {
  server_name kiosk.elitecollections.qa;
  root /var/www/elite/client-web;
  add_header Content-Security-Policy "default-src 'self'; frame-ancestors 'none';";
  location / { try_files $uri $uri/ /index.html; }
  location /api { proxy_pass http://localhost:3000; }
}
8
SSL certificate
Run: sudo certbot --nginx -d kiosk.elitecollections.qa
Certbot automatically edits the Nginx config and adds the HTTPS server block. Certificate is free (Let's Encrypt) and auto-renews every 90 days. Takes ~2 minutes.
9
Test the URL
Visit https://kiosk.elitecollections.qa/kiosk in a browser — the kiosk welcome screen should load. Test form submission goes into the DB. Test auto-reset timer.
Phase 3 — iPad physical setup
Lock the iPad to kiosk mode
Recommended: iOS Guided Access (free, built-in)
No third-party app needed. Locks the iPad to a single app and disables gestures. Staff set a PIN. Customers cannot leave the kiosk page.
10
Open kiosk URL in Safari
Open https://kiosk.elitecollections.qa/kiosk in Safari on the iPad. Tap Share → Add to Home Screen. Name it "Elite Feedback". Now it launches as a fullscreen PWA — no browser chrome, no URL bar, no tabs.
11
Enable Guided Access
Settings → Accessibility → Guided Access → ON
Set a Passcode (staff only). Enable Face ID / Touch ID for ending sessions.

Then: open the Elite Feedback app → triple-click the Home button (or Side button on newer iPads) → tap Start. The iPad is now locked to the kiosk. Customers can tap within the app but cannot go home, use other apps, or access settings.
12
Disable auto-lock during kiosk hours
Settings → Display & Brightness → Auto-Lock → Never
Prevents the screen from going dark between customers. Re-enable at the end of business hours if leaving the iPad unattended overnight.
13
(Optional) Apple Configurator for multiple iPads
If you have 2+ iPads, use Apple Configurator 2 (free Mac app) to deploy Single App Mode via MDM profile. This is the enterprise-grade version of Guided Access — it automatically locks the device to the kiosk app on boot, even after restart.
